    Example Usage

    The PRF is the pixel response function. This is used to define where flux from a star will fall. The PRF reference product is given in pandora-ref. This package enables you to use those files to make a PRF object.

    There are three kinds of PRFs currently implemented:

    • PRF this base class has a simple PRF
    • SpatialPRF this class enables the PRF shape to vary as a function of position on the detector
    • DispersedPRF this class enables the PRF to be distributed along a trace.

    Let's make some PRFs so you can see how to use them

    In[1]:
    Copied! 
    import pandoraaperture as pa
import matplotlib.pyplot as plt

    PRFs have two key words which show where they should be evaluated on the detector

    imcorner is the tuple of the lower left corner of the image

    In[8]:
    Copied! 
    prf.imcorner
    prf.imcorner
    Out[8]:
    (0, 0)

    We see here this is set to (0, 0), for the visible detector this is the corner.

    There is also imshape which sets the shape of the detector.

    In[9]:
    Copied! 
    prf.imshape
    prf.imshape
    Out[9]:
    (2048, 2048)

    These set where the PRF is evaluated. So, if we want to change where the PRF is evaluated we should keep these tuples in mind.

    The default for evaluation is the middle of the detector.

    In[10]:
    Copied! 
    row, column, flux = prf.evaluate()

fig, ax = plt.subplots()
ax.pcolormesh(column, row, flux)
ax.set(aspect='equal', xlabel='Column', ylabel='Row', title='PRF Model');
    row, column, flux = prf.evaluate() fig, ax = plt.subplots() ax.pcolormesh(column, row, flux) ax.set(aspect='equal', xlabel='Column', ylabel='Row', title='PRF Model');

    But you can change where you evaluate by updating the location

    In[11]:
    Copied! 
    row, column, flux = prf.evaluate(location=(400, 1500))

fig, ax = plt.subplots()
ax.pcolormesh(column, row, flux)
ax.set(aspect='equal', xlabel='Column', ylabel='Row', title='PRF Model');
    row, column, flux = prf.evaluate(location=(400, 1500)) fig, ax = plt.subplots() ax.pcolormesh(column, row, flux) ax.set(aspect='equal', xlabel='Column', ylabel='Row', title='PRF Model');

    This hasn't changed the shape of the PRF, because it is a simple PRF. Let's use a spatially dependent one instead.

    Now the PRF has changed significantly because we are far from the center.

    These objects embed the PRF at the correct position and have the full extent of the image. They are sparse, so they don't take up a lot of memory. You can dot them with 1 to convert them to a dense array.

    In[35]:
    Copied! 
    fig, ax = plt.subplots()
ax.imshow(X.dot(1), vmin=0, vmax=0.001, origin='lower')
ax.set(aspect='equal', xlabel='Column', ylabel='Row', title=f'PRF Model');
    fig, ax = plt.subplots() ax.imshow(X.dot(1), vmin=0, vmax=0.001, origin='lower') ax.set(aspect='equal', xlabel='Column', ylabel='Row', title=f'PRF Model');

    PRFs are designed to work with the shape of a single target. SkyScene objects work with all the targets Pandora will see.
